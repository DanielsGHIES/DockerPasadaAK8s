# Práctica Final: Aplicación segura con Kubernetes + KICS

## Objetivos

El objetivo de esta práctica es:

1. Migrar una aplicación multicontenedor de Docker a una aplicación escalable gestionada por Kubernetes.
2. Analizar con KICS la aplicación para identificar y corregir vulnerabilidades a nivel de infraestructura.

## Requisitos de documentación

El proceso deberá documentarse con capturas de pantalla con las siguientes características:

- Deben ser de pantalla completa.
- Debe ser visible la fecha y hora de realización.
- No serán válidos los recortes.

## Estructura general de la práctica

1. Punto de partida: aplicación Docker multicontenedor funcional.
2. Migración de Docker a K8s escalable en un repositorio independiente.
3. Análisis de vulnerabilidades con KICS.
4. Entrega final en un único archivo Zip.

## Repositorios requeridos

- Repositorio 1: aplicación Docker.
- Repositorio 2: aplicación Kubernetes.

## Entregas parciales

- Enlace al repositorio con la aplicación Docker.
- Archivo Zip con la aplicación K8s sin analizar.
- Documento HTML correspondiente al primer análisis de vulnerabilidades.
- Enlace al repositorio con la aplicación Kubernetes.
